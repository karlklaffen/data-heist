================================================================================
SCALER ANALYSIS - DECISION & OBSERVATIONS
================================================================================

RECOMMENDATION: DO NOT USE StandardScaler
Reason: Scaler consistently DECREASES model performance across all algorithms

================================================================================
PERFORMANCE DEGRADATION WITH SCALER
================================================================================

Random Forest Regression:
  - R² Without Scaler: 0.0795
  - R² With Scaler:    0.0494
  - Degradation:       -37.9%

Linear Regression:
  - R² Without Scaler: 0.0264
  - R² With Scaler:    0.0188
  - Degradation:       -28.8%

Ridge Regression:
  - R² Without Scaler: 0.0264
  - R² With Scaler:    0.0188
  - Degradation:       -28.8%

CONCLUSION: StandardScaler hurts ALL models. Best model remains Random Forest
without scaler (R² = 0.0795)

================================================================================
DATA CHARACTERISTICS EXPLAINING WHY SCALER FAILS
================================================================================

1. WEAK SIGNAL
   - Max correlation (longitude): 0.1489
   - Max correlation (latitude): 0.0647
   - Interpretation: Features have poor relationship with targets
   - Impact: Scaler cannot improve weak signals; hurts tree algorithms

2. NARROW OUTPUT RANGE
   - Latitude range: 0.0846° (33.580° to 33.665°)
   - Longitude range: 0.0553° (-117.632° to -117.577°)
   - Interpretation: Predicting fine-grained geographic variation
   - Impact: Raw feature scales are already appropriate for the problem

3. GOOD SAMPLE:FEATURE RATIO
   - Samples: 8000
   - Features: 30
   - Ratio: 266.67x (well-resourced)
   - Impact: Complex models safe; scaler not needed for regularization

4. NON-LINEAR SIGNAL DETECTED
   - Random Forest significantly outperforms linear models
   - Inference: Data relationships are tree-interpretable, not linearly
   - Impact: StandardScaler assumes linear relationships; violates this data

================================================================================
WHY SCALER FAILED FOR THIS SPECIFIC PROBLEM
================================================================================

StandardScaler applies to weak signal + narrow range:
- Scaler normalizes features to ~mean=0, std=1 range
- For weak signals, this disrupts the already-fragile feature-target link
- For tree algorithms, scaler provides no benefit (trees scale-invariant)
- For narrow output range, rescaling inputs doesn't improve predictions

Typical scaler benefits (which DON'T apply here):
  ✗ Prevents large-scale features from overpowering small ones (features already similar scale)
  ✗ Helps with linear regression (signal is non-linear)
  ✗ Stabilizes optimization (only for gradient-based; trees are discrete)

================================================================================
DECISION RATIONALE
================================================================================

The scaler experiments were archived to preserve the DISCOVERY PROCESS:
1. Test: Train models both with and without scaler
2. Measure: Capture R² metrics for all combinations
3. Analyze: Compare performance deltas
4. Decide: Choose the better approach based on DATA, not assumptions

Result: Data characteristics demonstrated that scaler is counterproductive.
The lack of feature-target correlation combined with non-linear relationships
and tree-based classifiers makes feature normalization detrimental.

This analysis validates a key ML principle:
"Before adding complexity (like scalers), understand what the data actually needs."
